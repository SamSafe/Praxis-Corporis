# Architecture Decision Records (ADRs)

An ADR captures **one significant decision**: the context, the choice, and the
consequences. They're short, dated, append-only (supersede rather than edit), and they
save you from re-litigating settled questions six months later.

## When to write one
Anything expensive to reverse: stack choices, 3D sourcing approach, licensing, the
id/naming convention, local-first vs backend, etc.

## Format
Copy `0000-template.md` to `NNNN-short-title.md` and fill it in.

## Index
- 0001 — _(pending)_ Choose 3D model sourcing approach
- 0002 — _(pending)_ Choose code license
- 0003 — _(pending)_ Confirm v1 tech stack
- 0004 — _(pending)_ Lock shared-id naming convention
